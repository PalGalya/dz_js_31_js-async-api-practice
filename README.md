# JavaScript HTTP Methods Testing Project

Проект для тестування основних HTTP-методів (GET, POST, PUT, PATCH, DELETE) з використанням JSONPlaceholder API.

## Опис проекту

Цей проект містить реалізацію та тестування п'яти основних HTTP-методів:

1. `getData()` - GET запити для отримання даних
2. `postData()` - POST запити для створення нових ресурсів
3. `putData()` - PUT запити для повного оновлення ресурсів
4. `patchData()` - PATCH запити для часткового оновлення ресурсів
5. `deleteData()` - DELETE запити для видалення ресурсів

## Технології

- JavaScript (ES6+)
- Vitest - для тестування
- JSONPlaceholder API
- Fetch API

## Встановлення

1. Клонуйте репозиторій:

```sh
git clone [url-репозиторію]
```

2. Встановіть залежності:

```sh
npm install
# або
yarn
```

## Запуск тестів

Для запуску тестів використовуйте команду:

```sh
npm test
# або
yarn test
```

Для запуску тестів у режимі спостереження:

```sh
npm run test:watch
# або
yarn test:watch
```

## API Документація

### getData(segment)

- **Параметри**: `segment` - рядок, який вказує на ендпоінт API
- **Повертає**: Promise з даними або повідомленням про помилку

### postData(segment, data)

- **Параметри**:
  - `segment` - рядок, який вказує на ендпоінт API
  - `data` - об'єкт з даними для відправки
- **Повертає**: Promise з відповіддю сервера або повідомленням про помилку

### putData(id, data)

- **Параметри**:
  - `id` - ідентифікатор ресурсу
  - `data` - об'єкт з даними для оновлення
- **Повертає**: Promise з оновленими даними або повідомленням про помилку

### patchData(id, data)

- **Параметри**:
  - `id` - ідентифікатор ресурсу
  - `data` - об'єкт з частковими даними для оновлення
- **Повертає**: Promise з оновленими даними або повідомленням про помилку

### deleteData(id)

- **Параметри**: `id` - ідентифікатор ресурсу для видалення
- **Повертає**: Promise з булевим значенням успішності операції
